The Anthropic API only accepts these exact strings for media_type:

    image/webp

If mimeType is arriving as something else — image/jpg, image/jpe, undefined getting coerced wrong, or any other value — Anthropic returns a 400. The || 'image/jpeg' fallback only fires if mimeType is falsy. If it's a non-empty wrong string, the fallback never fires.
